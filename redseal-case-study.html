Overview

A powerful tool that nobody could use efficiently

RedSeal is a cloud security intelligence platform used by cloud engineers, network administrators, and CISOs to visualise, monitor, and protect complex cloud infrastructure. The product worked — but the experience was cluttered, overwhelming, and inconsistent across different user roles.

There was no structure to prioritise key workflows. Different users were expected to navigate the same dense interface regardless of their role — creating confusion, inefficiency, and a growing gap between RedSeal's positioning as a user-first platform and the actual experience it delivered.

RedSeal wanted to re-establish itself as an industry leader with a focus on UX — not just capability. That meant rebuilding the experience around the needs of distinct user personas, not around the product's feature set.

Research

Understanding three very different users

🔍

Heuristic evaluation

A systematic review of the existing system revealed redundant workflows, poor task visibility, and inconsistent navigation patterns across modules.

👥

Persona interviews

Spoke with cloud engineers, network admins, and CISOs — each with dramatically different goals, technical depth, and daily task patterns that the existing UI failed to accommodate.

🗺

Task mapping workshops

Collaborated with the RedSeal client team to identify the most frequently used features per persona — building an evidence-based foundation for redesign priorities.

🧱

Core finding

Users struggled to locate resources, track task progress, and navigate their cloud infrastructure at scale. The IA was optimised for features, not for tasks.

Problem Framing

Four design challenges

"How might we simplify a technically dense security product into a flexible, persona-specific UX system — without sacrificing the depth that power users depend on?"

Information Architecture

Restructure the navigation and IA to prioritise task flows over feature lists.

Infrastructure Visualisation

Help users understand the hierarchy and connections in complex cloud environments at a glance.

Resource Monitoring

Simplify how users track and prioritise important resources across their infrastructure.

Task Tracking

Give users a centralised, clear view of task status and progress across the platform.

Reflection

Key learnings

What worked well

  • Persona-led design gave us clarity on what to prioritise — every feature was evaluated through the lens of a specific user's goals
  • Task mapping workshops with the client team surfaced real usage patterns that desktop research would have missed
  • The modular design approach allowed us to redesign core modules independently without breaking the product's overall coherence
  • Mentoring junior designers throughout gave the whole team shared context and raised output quality

What I'd do differently

  • Measure baseline task completion rates per persona before redesign to create a clearer before/after story
  • Run usability testing with real users mid-design — not just walkthroughs — to catch navigation issues earlier
  • Build a shared component library earlier in the project to maintain consistency as the team scaled across modules
